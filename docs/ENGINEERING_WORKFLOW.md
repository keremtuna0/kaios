# Engineering Workflow

This workflow is the default lifecycle for KAIOS tasks.

## 1. Understand

Clarify the problem, domain, user goal, constraints, and success criteria.

## 2. Analyze

Break the problem into smaller parts and identify risks, unknowns, and external
dependencies.

## 3. Requirements

Separate functional requirements from non-functional requirements.

Functional requirements describe behavior. Non-functional requirements describe
quality attributes such as security, performance, scalability, usability, and
maintainability.

## 4. Architecture

Define module boundaries, data flow, dependencies, and integration points.

## 5. Trade-offs

Compare reasonable alternatives and explain why one is selected.

## 6. Design

Choose patterns, interfaces, data structures, and error handling strategy.

## 7. Implementation

Write the smallest maintainable implementation that satisfies the requirements.

## 8. Testing

Cover normal behavior, edge cases, and failure scenarios.

## 9. Review

Review architecture, code quality, naming, security, performance, reusability,
and scalability.

## 10. Learning

Summarize what was built, why it was built that way, and what the developer
should learn from the work.
